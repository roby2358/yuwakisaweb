Random skill template:

  - Pick a damage formula: e.g. (statA × N) + (statB × M), or (statA + statB) × scale
  - Roll a bonus effect from a pool — AoE-1, knockback, bleed/DoT, stun, lifesteal, armor-pierce, chain-to-adjacent, mark-for-bonus-damage, terrain-aware multiplier, etc.

  - Aether cost rolled in the standard tier (≤3 cheap, 4-5 mid, ≥6 finisher) keyed to power level

System pieces:
  - Random skill generator: takes a Rando, returns a runtime SKILL object with {id, name, scaling, cost, range, effect} matching the SKILLS table shape so the rest of the engine doesn't care if a skill came from the hand-crafted table or the RNG

  - Persistence: a rolled-and-kept random skill needs to survive save/load. Either serialize the full skill object on player.learnedSkills (currently a Set of string IDs — would need to widen) or seed-based regen so the same skill comes back from the same seed
  - Scope check: should random skills be offered alongside standard ones (player picks among 3 of each), or as a separate "wild" option (player rolls into a random one)?

Dress up the hex map with per-terrain visual character. The current green/brown/yellow/gray/blue palette is good — clean and readable — but flat. Add subtle decorative accents so each terrain feels like a place, not a colored cell.

Two approaches (probably both):
  1. Sprite-sheet additions: extend or add a new sheet (e.g. `terrainSheet`) with small per-terrain icons — trees for forest, rocks/peaks for hills/mountains, reeds for marsh, ripples or waves for water, ruins/cracks for shattered/distressed hexes, grass tufts for plains. SpriteSheet.drawCell already handles tinting so terrain palette can still drive color.
  2. Drawn accents: cheap programmatic canvas shapes — small triangles for mountains, dots for sand, wavy lines for water, jagged scribbles for chaos hexes. No asset pipeline needed.

Design constraints:
  - Tactical readability stays paramount. A decorated forest hex must still be obviously "forest" at a glance and not be confused with hills. Keep accents low-contrast and small relative to the hex.
  - Determinism: per-hex accent variant should be picked from a Rando seeded on (q, r) so the map looks the same every render and across save/load — no flicker, no churn.
  - Density: don't decorate every hex; some hexes plain, some with one accent, rare hexes with a "feature" sprite (a bigger tree, a boulder cluster). Variation > uniformity.
  - Fog/visibility: accents fog and reveal alongside the terrain layer; revealed-but-not-visible accents desaturate same as everything else.
  - Don't conflict with POI tokens, enemy tokens, or highlight overlays — accents draw beneath those layers.

Implementation: extend the render pipeline in index.js's hex draw step, after terrain fill and before tokens/highlights. Probably a new helper like `drawTerrainAccent(ctx, hex, x, y)` that picks a deterministic variant from a per-terrain table. New sprite sheet PNG if going the sprite route.

Stretch: chaos/shattered hexes get animated-feeling accents (jittered cracks redrawn each frame) — but that breaks determinism and could be a separate task.
