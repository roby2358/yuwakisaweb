Break the "dump everything into warding" dominant strategy by reshaping the skill roster around two tiers, AND repassing aether costs at the same time.

Today the power trio (cosmic bolt, starfall, meteor) all scale ~3× warding, which double-dips with warding's role as the aether pool — at warding=27 each cast hits 81, dwarfing melee, and the aether costs are low enough to spam them.

Two-tier fix:

(1) Add new combo skills — scale on warding + a second stat, cheaper aether. These become the bread-and-butter rotation, and each is best for a different stat investment so optimizing one stat no longer optimizes everything.
  - Cosmic bolt → combo: warding × 2 + reflex (precision burst, single-target king)
  - Add new combat-flavored combos: warding + might (raw force), warding + vigor (sustain/heal-on-hit)
  - Combo skills cost ~2-4 AE so they're sustainable

(2) Keep meteor and starfall as pure-warding nukes, but make them expensive — "saved up for this" moments, not per-turn rotation. Warding pool still funds them; scarcity not formula does the balancing.
  - Starfall — keep ~3× warding AoE (the ruin-popper stays brutal), aether cost ~6
  - Meteor — keep 3× warding AoE, aether cost ~5

Aether-cost calibration: +3 AE magic items are common, so 3 AE is effectively the "break even" cost (an item refund covers it). Anything cheap enough to spam should be ≤3 AE; anything that's supposed to be a finisher should be ≥5 AE so an item proc doesn't fully refund it.

Audit the rest of SKILLS for any other pure-warding scalers and either give them a thematic second stat (combat → might, precision → reflex, healing/sustain → vigor) or move them to the expensive-nuke tier.

Pairs with warrior-4mfy: Land Health items reward non-warding builds, and this fix makes those builds actually competitive instead of strictly worse. Also pairs with might-def-pmtb (melee survivability buff).

Tradeoff: don't nerf in isolation. Tune so a balanced-stat caster lands near today's pure-warding output, and a pure-warding 27 build still has the burst nukes but can't sustain them. Update SKILLS damage formulas + aether costs in config.js (or wherever skill resolution lives in index.js); retest end-game encounters.

When the player gets to pick a skill (level-up via pendingSkillChoice, or possibly at ruins/POIs), roll first: with some probability pick a standard skill from SKILLS as today, otherwise roll a randomized attack skill instead.

Random skill template:

  - Pick a damage formula: e.g. (statA × N) + (statB × M), or (statA + statB) × scale
  - Roll a bonus effect from a pool — AoE-1, knockback, bleed/DoT, stun, lifesteal, armor-pierce, chain-to-adjacent, mark-for-bonus-damage, terrain-aware multiplier, etc.
  - Generate a flavorful name (template-driven, e.g. "[Adjective] [Noun]" pulled from word pools, or stat-themed: might+reflex → "Searing Strike", warding+vigor → "Aether Bulwark Bolt")
  - Aether cost rolled in the standard tier (≤3 cheap, 4-5 mid, ≥6 finisher) keyed to power level

System pieces:
  - Random skill generator: takes a Rando, returns a runtime SKILL object with {id, name, scaling, cost, range, effect} matching the SKILLS table shape so the rest of the engine doesn't care if a skill came from the hand-crafted table or the RNG
  - Skill-pick site: at level-up dialog (and any other skill-pick UI), roll once; if random-skill, generate one and offer it instead of (or alongside) a standard option

  - Scope check: should random skills be offered alongside standard ones (player picks among 3 of each), or as a separate "wild" option (player rolls into a random one)?

Design feel: every run gets weird, unique tools. Pairs with the broader design principle of "discoverable rule-breaking combos are fun" — random skills + items + stat builds = more degrees of freedom for the player to find a busted combo.

Dress up the hex map with per-terrain visual character. The current green/brown/yellow/gray/blue palette is good — clean and readable — but flat. Add subtle decorative accents so each terrain feels like a place, not a colored cell.

Two approaches (probably both):
  1. Sprite-sheet additions: extend or add a new sheet (e.g. `terrainSheet`) with small per-terrain icons — trees for forest, rocks/peaks for hills/mountains, reeds for marsh, ripples or waves for water, ruins/cracks for shattered/distressed hexes, grass tufts for plains. SpriteSheet.drawCell already handles tinting so terrain palette can still drive color.
  2. Drawn accents: cheap programmatic canvas shapes — small triangles for mountains, dots for sand, wavy lines for water, jagged scribbles for chaos hexes. No asset pipeline needed.

Design constraints:
  - Tactical readability stays paramount. A decorated forest hex must still be obviously "forest" at a glance and not be confused with hills. Keep accents low-contrast and small relative to the hex.
  - Determinism: per-hex accent variant should be picked from a Rando seeded on (q, r) so the map looks the same every render and across save/load — no flicker, no churn.
  - Density: don't decorate every hex; some hexes plain, some with one accent, rare hexes with a "feature" sprite (a bigger tree, a boulder cluster). Variation > uniformity.
  - Fog/visibility: accents fog and reveal alongside the terrain layer; revealed-but-not-visible accents desaturate same as everything else.
  - Don't conflict with POI tokens, enemy tokens, or highlight overlays — accents draw beneath those layers.

Implementation: extend the render pipeline in index.js's hex draw step, after terrain fill and before tokens/highlights. Probably a new helper like `drawTerrainAccent(ctx, hex, x, y)` that picks a deterministic variant from a per-terrain table. New sprite sheet PNG if going the sprite route.

Stretch: chaos/shattered hexes get animated-feeling accents (jittered cracks redrawn each frame) — but that breaks determinism and could be a separate task.
